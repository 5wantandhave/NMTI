## 二、玩法机制

### 2.1 核心流程

```
首页引导 → 答题(16题) → 计算结果 → 雷达图展示 → 逆天改命
```

### 2.2 答题规则

- **场景分组**：
  - 任务执行（1-4题）
  - 社交博弈（5-8题）
  - 突发压力（9-12题）
  - 利益分配（13-16题）

### 2.3 评分系统 - 常模均值切割法

#### 4维度连续体

| 维度代码 | 维度名称 | 高分 = 左侧字母 | 低分 = 右侧字母 |
|----------|----------|-----------------|-----------------|
| **J/P** | 卷摸维度 | J = 终极卷王 | P = 终极摸鱼 |
| **I/E** | 耗放维度 | I = 重度内耗 | E = 疯狂外放 |
| **F/T** | 付理维度 | F = 纯纯大冤种 | T = 绝对冷酷理智 |
| **S/N** | 实幻维度 | S = 脚踏实地 | N = 天天幻想躺平 |

#### 常模均值（社会基准）

| 维度 | 常模均值 μ | 说明 |
|------|-----------|------|
| JP | +2 | 社会普遍偏卷 |
| IE | +1 | 职场人普遍内耗 |
| FT | 0 | 感性理智各半 |
| SN | +2 | 打工被迫务实 |

#### 计分公式

```
用户原始分 X → 计算与常模μ的偏差 → 与16种类型进行匹配度计算
```

#### 类型匹配算法（修复后）

```javascript
// 1. 计算偏差值
deviations = {
  JP: JP - μ_JP,
  IE: IE - μ_IE, 
  FT: FT - μ_FT,
  SN: SN - μ_SN
}

// 2. 为每种类型定义期望偏差值
typeExpectedDeviations = {
  IJFS: { JP: +1, IE: -1, FT: +1, SN: +1 },
  IJFN: { JP: +1, IE: -1, FT: +1, SN: -1 },
  // ... 其他14种类型
}

// 3. 计算匹配度（绝对差之和）
matchScore = Σ|用户偏差 - 类型期望偏差|

// 4. 选择匹配度最高的类型
bestMatch = argmin(matchScore)
```

**算法优势**：
- 避免简单的阈值判断，更准确地反映用户真实性格
- 支持更精细的类型区分
- 基于4个维度得分与所有类型进行匹配，而非一刀切

## 四、功能亮点

### 4.1 雷达图展示
- SVG 绘制四维雷达图
- 直观展示用户在卷/内耗/付出/务实四个维度的得分
- 显示原始分数（+分/-分）

### 4.2 逆天改命（修复后）
- **双向进化逻辑**：为每个维度提供双向进化建议
- **智能匹配**：根据当前类型自动选择对应的进化方向
- **合理阈值**：偏差绝对值≥1即生成进化建议

#### 进化方向示例
| 当前类型 | 进化建议 | 目标类型 |
|----------|----------|----------|
| J类型 | 少回一个"收到" | 向P类型进化 |
| P类型 | 多回一个"收到" | 向J类型进化 |
| I类型 | 少一点精神内耗 | 向E类型进化 |
| E类型 | 多一点社交互动 | 向I类型进化 |
| F类型 | 少当一次大冤种 | 向T类型进化 |
| T类型 | 多一点人情味 | 向F类型进化 |
| S类型 | 少一点务实精神 | 向N类型进化 |
| N类型 | 多一点脚踏实地 | 向S类型进化 |

- 例如："如果你当初选了B，你将从 996死者卷王 进化为 精准摸鱼学家"

### 4.3 详细类型解读

## 八、Bug修复记录

### 修复时间：2026-04-11

#### 修复问题1：类型匹配算法错误
**原问题**：使用固定阈值判断类型（JP≥2→J，IE≥1→I，FT≥0→F，SN≥2→S）
**修复方案**：实现基于4个维度得分与所有类型进行匹配的算法
**效果**：更准确地反映用户真实性格类型

#### 修复问题2：逆天改命功能错误
**原问题**：只检查正偏差，忽略负偏差；只考虑单向进化
**修复方案**：实现双向进化逻辑，为每个维度提供双向进化建议
**效果**：根据当前类型智能生成针对性的进化路径
